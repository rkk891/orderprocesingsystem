Reliability controls

State changes remain valid under concurrent work.

Transactions protect complete aggregates. Conditional PostgreSQL updates ensure competing workers cannot overwrite a newer state.

Transaction safety

All items commit together.

Creating an order saves the parent and every item in one transaction. Any failure rolls the complete aggregate back.

Concurrency safety

Two workers, one valid winner.

Every mutation includes the expected current status in its database update. A zero-row result becomes a clear missing-or-conflict decision.
